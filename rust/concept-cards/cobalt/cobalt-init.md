# Quick Definition

`cobalt init` creates a new Cobalt site with example pages, posts, and layouts, either in the current directory or in a specified sub-directory.

# Core Definition

The `cobalt init` command scaffolds a new Cobalt project by generating the standard directory structure including example pages, posts, and layouts. It can be invoked in the current directory (`cobalt init`) or with a target directory argument (`cobalt init myBlog`) to create the site in a sub-directory (source: Usage doc page, "init" section). This is the standard first step after installing Cobalt, producing a working site that can immediately be served or built.

# Prerequisites

- **Installation** -- Cobalt must be installed before running `cobalt init`

# Key Properties

1. **Scaffolds a complete site** -- "Create a site with an example pages, posts, and layouts" (source: Usage doc page, "init" section).
2. **Optional directory argument** -- can target the current directory or a named sub-directory (source: Usage doc page).
3. **Generates standard structure** -- creates the directory layout including `_cobalt.yml`, `_layouts`, `_includes`, `posts`, and other conventional directories (inferred from Directory doc page).
4. **Idempotent starting point** -- produces a site ready for immediate `cobalt serve` or `cobalt build`.

# Construction / Recognition

## To Construct/Create:
1. Navigate to the desired parent directory.
2. Run `cobalt init` to initialize in the current directory, or `cobalt init myBlog` to create a sub-directory.

## To Identify/Recognize:
1. A successfully initialized Cobalt site contains a `_cobalt.yml` file and the standard underscore-prefixed directories.
2. The init command produces example content files that can be built immediately.

# Context & Application

- **Typical contexts**: Starting a new blog or static website project with Cobalt.
- **Common applications**: Bootstrapping a new site, creating a project for experimentation or learning Cobalt.

# Examples

**Example 1** (source: Usage doc page): Initialize in the current directory:
```console
$ # Create site in current directory
$ cobalt init
```

**Example 2** (source: Usage doc page): Initialize in a sub-directory:
```console
$ # Create site in `myBlog` sub-directory
$ cobalt init myBlog
```

**Example 3** (source: Getting Started page): Full workflow starting from init:
```console
$ mkdir myBlog && cd myBlog
$ cobalt init
```

# Common Errors

- **Error**: Running `cobalt init` in a directory that already contains a Cobalt site.
  **Correction**: Use an empty directory or specify a new sub-directory name as an argument.

# Common Confusions

- **Confusion**: Believing `cobalt init` only works with a directory argument.
  **Clarification**: `cobalt init` can be run without arguments to initialize the current directory, or with a directory name to create and initialize a sub-directory.
