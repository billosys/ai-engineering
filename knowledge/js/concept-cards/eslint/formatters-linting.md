# Quick Definition
An ESLint formatter (linting context) is a package that controls the appearance of linting results in the CLI, presenting the report of violations to the user.

# Core Definition
A formatter in the ESLint linting context is a package that presents the report generated by ESLint. It controls how violations are displayed to the user in CLI output. ESLint ships with several built-in formatters, including `stylish` (the default), `json`, and `html`. Custom formatters can also be created using the same Formatter API as built-in formatters.

# Prerequisites
- eslint: Formatters present ESLint's output
- violations: Formatters display the collected violations

# Key Properties
1. **Output presentation** -- Controls how linting results appear in the CLI
2. **Built-in options** -- Includes `stylish` (default), `json`, and `html`
3. **Pluggable** -- Custom formatters follow the same API as built-in ones
4. **Report-based** -- Receives the complete report of violations to format

# Construction / Recognition
- Specified via CLI: `eslint --format json .`
- Default is `stylish` if not specified
- Custom formatters can be loaded from npm packages

# Context & Application
Formatters are used to tailor ESLint output for different consumers. The `stylish` formatter is readable for developers in a terminal. The `json` formatter is useful for programmatic consumption, CI integrations, or custom tooling. The `html` formatter can generate reports for web-based dashboards.

# Common Errors
1. Confusing ESLint formatters (output presentation) with code formatters (Prettier, dprint) -- these are entirely different concepts

# Common Confusions
1. **Formatter (linting) vs. formatter (tool)** -- An ESLint formatter controls CLI output appearance; a code formatter (like Prettier) reformats source code. The glossary explicitly distinguishes these two meanings
