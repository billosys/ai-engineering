Charter

Charter was designed by Matthew Carter in 1987 as a <a href="body-text.html" class="xref">body text</a> font that would hold up well on low-resolution output devices of the day—fax machines and 300 dpi laser <span class="no-hyphens">printers.</span>

Though 300 dpi printing is long gone, most of today’s computer displays operate far below 300 dpi. Charter, unsurprisingly, holds up admirably well as a screen font. (Matthew would go on to design the quintessential screen fonts <a href="verdana-alternatives.html" class="xref">Verdana</a> and <a href="georgia-alternatives.html" class="xref">Georgia</a> for <span class="no-hyphens">Microsoft.)</span>

In 1992, Bitstream contributed PostScript versions of the basic Charter family to the [X Consortium.](http://www.x.org/wiki/XConsortium/) These original files are still available and can be freely downloaded and modified. But they can be hard to find. So I tracked them down and converted them into a new package of OTFs, TTFs, and <span class="no-hyphens">webfonts.</span>

Why? Because Charter is a great font, and easily one of the best free fonts available. So I’m doing what I can to put more Charter into the world. If your project demands a font without licensing restrictions, consider <span class="no-hyphens">it.</span>

<div class="btw-title">

by the way

</div>

- If you run a recent version of Mac OS, Charter is already included as a <a href="system-fonts.html" class="xref">system font</a>. So if you only want to use it on your own machine, you don’t need to download this <span class="no-hyphens">version.</span>

- Charter is also available as a full [OpenType font family](http://www.myfonts.com/fonts/bitstream/charter/) (though not for <span class="no-hyphens">free).</span>

- Charter is the basis of another free font called [Charis.](http://scripts.sil.org/cms/scripts/page.php?site_id=nrsi&id=charissil_download) Charis has a larger character set, so if you need that, you might prefer it to this version. But if not, stick with this one—Charis includes some questionable design <span class="no-hyphens">modifications.</span>
