research papersBigger margins, smaller point size, tighter line spacing

Does this look familiar? A document with one-inch margins on all sides, 12-point font, and double-spaced lines? These were idiomatic <a href="typewriter-habits.html" class="xref">typewriter habits</a>. That’s why they became the basis of many institutional document-layout rules. Nearly 40 years into the digital-typesetting era, they remain with <span class="no-hyphens">us.</span>

<a href="samples/research-paper-before.pdf" class="sample-document"></a>

Hat tip to [SCIgen](http://pdos.csail.mit.edu/scigen/) for the text in this <span class="no-hyphens">example.</span>

But have you ever seen a book, newspaper, or magazine that uses this layout? No. Why not? Because it’s not optimally legible. So why would anyone use it? Because it suits the severely limited capabilities of the typewriter. So if we don’t use typewriters anymore, why does everyone still use this <span class="no-hyphens">layout?</span>

My thoughts <span class="no-hyphens">exactly.</span>

<div id="before" class="subhead">

[Before](#before)

</div>

1.  Quick review—the first four items on this list determine how the <a href="body-text.html" class="xref">body text</a> looks, and are therefore <span class="no-hyphens">important.</span>

    <a href="page-margins.html" class="xref">Page margins</a> too small; <a href="line-length.html" class="xref">Line length</a> too <span class="no-hyphens">wide.</span>

2.  <a href="point-size.html" class="xref">Point size</a> too <span class="no-hyphens">big.</span>

3.  <a href="line-spacing.html" class="xref">Line spacing</a> too <span class="no-hyphens">tall.</span>

4.  <a href="times-new-roman-alternatives.html" class="xref">Times New Roman</a>—snore.

5.  Failure to put <a href="one-space-between-sentences.html" class="xref">one space between sentences</a>.

6.  <a href="first-line-indents.html" class="xref">First-line indents</a> too <span class="no-hyphens">deep.</span>

7.  <a href="justified-text.html" class="xref">Justified text</a> without <a href="hyphenation.html" class="xref">hyphenation</a>.

8.  <a href="underlining.html" class="xref">Underlining</a> in <span class="no-hyphens">headings.</span>

If some authority figure insists that you use this layout, then do so. If not, then don’t. It’s <span class="no-hyphens">awful.</span>

Fortunately, transplanting this document from the 1890s into the present is simple <span class="no-hyphens">surgery.</span>

<a href="samples/research-paper-after.pdf" class="sample-document"></a>

<div id="after" class="subhead">

[After](#after)

</div>

1.  Page margins larger; line length shorter (about 65 characters per <span class="no-hyphens">line).</span>

2.  Point size <span class="no-hyphens">smaller.</span>

3.  Line spacing <span class="no-hyphens">reduced.</span>

4.  Better fonts ( and )

5.  One space between <span class="no-hyphens">sentences.</span>

6.  Hyphenation turned <span class="no-hyphens">on.</span>

7.  No <span class="no-hyphens">underlining.</span>

<div class="btw">

<div class="btw-title">

by the way

</div>

- Another virtue of the revised layout: it fits more text on the <span class="no-hyphens">page.</span>

- Well, maybe that’s not always a virtue. One likely reason for the tenacity of the typewriter-style layout is that it produces a consistent number of words per page, preventing writers from abusing page limits. But in the digital age, the better way of controlling document length is with word counts. (See <a href="point-size.html" class="xref">point size</a> for <span class="no-hyphens">more.)</span>

- If you’re working on draft documents with others, you have to be careful about fonts—if you pick a font your collaborators don’t have, they won’t see the formatting accurately. This is one of the few situations where <a href="system-fonts.html" class="xref">system fonts</a> are your best choice. Your collaborators are likely to have them, and these fonts look good on screen, where much of the collaboration happens. If you like, you can reformat with a different font at the <span class="no-hyphens">end.</span>

  If it’s critical that your document appear the same way on your collaborator’s screen as it does on yours, the only foolproof technique is to share PDF files and use PDF-based commenting and review <span class="no-hyphens">tools.</span>
