Typography in ten minutes

This is a bold claim, but I stand behind it: if you learn and follow these five typography rules, you will be a better typographer than nearly every writer—and even most graphic <span class="no-hyphens">designers.</span>

All it takes is ten minutes: five minutes to read these rules once. Then five minutes to read them <span class="no-hyphens">again.</span>

**Small but necessary interruption**: This online book isn’t supported by banner ads. It’s supported by readers like you. If you find these tips valuable, see <a href="how-to-pay-for-this-book.html" class="xref">how to pay for this book</a>. There’s plenty of other material too—see the <a href="index.html" class="xref">table of contents</a>.

Ready? <span class="no-hyphens">Go.</span>

1.  The typographic quality of your document is determined largely by how the <a href="body-text.html" class="xref">body text</a> looks. Why? Because there’s more body text than anything else. So start every project by making the body text look good. Then worry about the <span class="no-hyphens">rest.</span>

    In turn, the appearance of the body text is determined primarily by these four typographic <span class="no-hyphens">choices:</span>

2.  <a href="point-size.html" class="xref">Point size</a> is how big the font is. In print, the most comfortable range for body text is **10–12 point**. On the web, **15–25 pixels**. Not every font appears equally large at a given point size, so be prepared to adjust as <span class="no-hyphens">needed.</span>

3.  <a href="line-spacing.html" class="xref">Line spacing</a> is the vertical distance between lines. It should be **120–145%** of the point size. In word processors, use the “Exact” line-spacing option to achieve this. The default single-line option is too tight; the 1½-line option is too loose. In CSS, use the `line-height` property (preferably with a [unitless value,](https://stackoverflow.com/a/20818206) so 130% would become `1.3`).

4.  <a href="line-length.html" class="xref">line length</a> is the horizontal width of the text block. Line length should be an average of **45–90 characters** per line (use your word-count function) or **2–3 lowercase alphabets**, like <span class="no-hyphens">so:</span>

    <span style="white-space:pre">abcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyzabcd</span>

    In a printed document, this usually means <a href="page-margins.html" class="xref">page margins</a> larger than one inch (a <a href="typewriter-habits.html" class="xref">typewriter habit</a>). On a web page, it usually means preventing the text from flowing to the edges of the browser <span class="no-hyphens">window.</span>

5.  And finally, font choice. The fastest, easiest, and most visible improvement you can make to your typography is to ignore the fonts already loaded on your computer (known as <a href="system-fonts.html" class="xref">system fonts</a>) and the <a href="free-fonts.html" class="xref">free fonts</a> that inundate the internet. Instead, buy a professional font (like those found in <a href="font-recommendations.html" class="xref">font recommendations</a>). A professional font gives you the benefit of a professional designer’s skills without having to hire <span class="no-hyphens">one.</span>

    If that’s impossible, you can still make good typography with system fonts. But choose wisely. And never choose <a href="times-new-roman-alternatives.html" class="xref">Times New Roman</a> or <a href="helvetica-and-arial-alternatives.html" class="xref">Arial</a>, as those fonts are favored only by the apathetic and sloppy. Not by typographers. Not by <span class="no-hyphens">you.</span>

That’s it. As you put these five rules to work, you’ll notice your documents starting to look more like professionally published <span class="no-hyphens">material.</span>

If you’re ready for a lot more, start at the <a href="foreword.html" class="xref">foreword</a> and keep <span class="no-hyphens">reading.</span>
