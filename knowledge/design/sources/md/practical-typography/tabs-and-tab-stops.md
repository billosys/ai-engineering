tabs and tab stopsFor horizontal space in the middle of a line

On typewriters, the tab key moved the carriage to a fixed horizontal position, marked with a tab stop. This allowed typists to create columns of text or numbers, also known as tabular layouts (hence the name *tab*).

Tabs and tab stops still work the same way. A tab stop marks a location; typing a tab moves the cursor to that <span class="no-hyphens">location.</span>

|     | Word | Pages | HTML          |
|-----|------|-------|---------------|
| tab | tab  | tab   | not available |

These days, the tab is used only for inserting horizontal space in the middle of a line. If you need horizontal space at the beginning of a paragraph, adjust the <a href="first-line-indents.html" class="xref">first-line indent</a>. For a true tabular layout, use a <a href="tables.html" class="xref">table</a>, not <span class="no-hyphens">tabs.</span>

Word and Pages do not display their default tab stops on the ruler (for no good reason). Be aware that adding your own tab stops wipes out the default ones. So you’ll usually want to set tab stops before inserting tabs in the <span class="no-hyphens">text.</span>

The tab is not as vital as it once was, but word processors still shortchange its capabilities. A new word-processing document has default tab stops every half inch. These default tab stops exist so that something happens when you type a tab in the new document. But this default behavior also suggests that what the tab key does is move the cursor a half inch at a time. Not <span class="no-hyphens">true.</span>

To get the most out of tabs, you should set your own tab stops. Avoid relying on the default tab stops—they undermine the goals of control and predictability. As with <a href="word-spaces.html" class="xref">word spaces</a>, also avoid using sequences of tabs to move the cursor around the screen. <span class="no-hyphens"></span>

To see your tab stops, display the <span class="no-hyphens">ruler.</span>

<div class="howto">

<div class="howto-name">

How to display the ruler

</div>

WordView → Show panel → Ruler

Mac OS WordView → Ruler

PagesView → Show Ruler, or type ⌘ + <span class="no-hyphens">r</span>

</div>

With the ruler visible, you can edit your tab <span class="no-hyphens">stops.</span>

<div class="howto">

<div class="howto-name">

How to edit tab stops

</div>

1.  To insert a new tab stop, click in the ruler where you want the tab <span class="no-hyphens">stop.</span>

2.  To move a tab stop, click and drag it in the <span class="no-hyphens">ruler.</span>

3.  To change a tab stop from one kind to another, double-click it in the ruler. In Pages, you can also right-click <span class="no-hyphens">it.</span>

4.  To remove a tab stop, drag it off the <span class="no-hyphens">ruler.</span>

</div>

Many layout tasks traditionally handled by tabs are now better handled by <a href="tables.html" class="xref">tables</a>. For instance, this illustration of tab stops was made with tabs, but it would’ve been easier to use a four-column, four-row <span class="no-hyphens">table.</span>

The default tab stop is a left tab stop, meaning text under the tab stop aligns to its left edges. Word processors also offer center and right tab stops, as well as decimal tab stops that align columns of numbers at their decimal points (see <a href="grids-of-numbers.html" class="xref">grids of numbers</a> for why this <span class="no-hyphens">matters).</span>

Use the proper tab stop for the job. For instance, don’t use a center tab stop to line up decimal numbers. A right tab can be useful in a document footer to put two pieces of text (e.g., the document title and the page number) at opposite ends of the same <span class="no-hyphens">line.</span>

Tabs are used in <a href="bulleted-and-numbered-lists.html" class="xref">bulleted and numbered lists</a> to separate the bullet or number from the text. Tabs are also used in automatically generated tables of contents and tables of authorities to put the page numbers at the right edge of the <span class="no-hyphens">table.</span>

Despite their names, automatic tables of contents and tables of authorities are not made with true word-processor tables. If you want to edit them, you need to be comfortable adjusting tab <span class="no-hyphens">stops.</span>

Tabs and tab stops have their place, but in most cases they act as a less-capable alternative to a <a href="tables.html" class="xref">table</a>. Use tabs and tab stops if your formatting task is truly simple. If not, upgrade to a <span class="no-hyphens">table.</span>

<div class="btw">

<div class="btw-title">

by the way

</div>

- You can fill the space in front of a tab (for instance, with periods or underscores) by using a *tab leader*. Don’t approximate this effect by typing a hundred periods or underscores manually in front of a tab. You won’t like the <span class="no-hyphens">consequences.</span>

  In HTML, there are no tabs or tab stops. But HTML does support tables. CSS layout modes called [flexbox and grid](https://rachelandrew.co.uk/archives/2016/03/30/should-i-use-grid-or-flexbox/) can be used to arrange larger collections of elements in tabular <span class="no-hyphens">fashion.</span>
