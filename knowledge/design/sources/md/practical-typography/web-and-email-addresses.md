web & email addressesDon’t hyphenate

Web addresses identify a location on the internet. They usually look like http://www.somelongname.com/folder/subfolder/page.html. Email addresses usually take the form <span class="no-hyphens">nameofperson@somelongname.com.</span>

In web pages, web and email addresses are usually hidden from view behind hyperlinks, so they don’t cause typographic <span class="no-hyphens">trouble.</span>

But in print, web addresses present two <span class="no-hyphens">problems.</span>

The first problem: web addresses can be long. Really, really long. Running the whole web address may be fine if you can bury it in a footnote. But it’s useless if you’re hoping readers will type the address on their <span class="no-hyphens">own.</span>

One problem with shortening services is that there’s no guarantee they’ll last as long as your document. Though that’s also true of the material at the underlying link. If you need to guarantee permanence: print <span class="no-hyphens">it.</span>

For a more usable web address, use an address-shortening service like [TinyURL](http://tinyurl.com) or [Bitly.](http://bitly.com) These services take a web address of any length and convert it into a short address like http://tinyurl.com/p5wf3c. This is easier to read and type. But it doesn’t reveal the underlying web address. It also isn’t guaranteed to work <span class="no-hyphens">permanently.</span>

If you put a web address in a footnote or endnote, consider running the long version with a shortened version next to it. Then you’re covered. For <span class="no-hyphens">instance:</span>

<div class="indented">

*See* http://www.cacd.uscourts.gov/CACD/LocRules.nsf/  
a224d2a6f8771599882567cc005e9d79/043e35b5803a6d3f  
8825768d0067b375?OpenDocument, *also available at* http://tinyurl.com/y6o4yte.

</div>

The second problem: web addresses are difficult to wrap onto multiple lines. A web address is one unbroken string of characters. You don’t want your web address hyphenated, because readers will likely mistake the hyphens for part of the address. Therefore, use <a href="hard-line-breaks.html" class="xref">hard line breaks</a> to set the points where the web address should wrap onto the next <span class="no-hyphens">line.</span>

Email addresses are shorter than web addresses and thus not as painful. But they shouldn’t be hyphenated either, for the same <span class="no-hyphens">reasons.</span>

Word processors have an annoying default habit of making every web and email address underlined and blue. That might make some sense if you’re creating a PDF that needs to include hyperlinks. But it makes no sense at all if you’re creating a document that needs to be <span class="no-hyphens">printed.</span>

<div class="howto">

<div class="howto-name">

How to turn off automatic hyperlinks

</div>

WordFile → Options → Proofing → AutoCorrect Options → AutoFormat As You Type → uncheck Internet and network paths with <span class="no-hyphens">hyperlinks</span>

Mac OS WordWord → Preferences → AutoCorrect → AutoFormat As You Type → uncheck Internet and network paths with <span class="no-hyphens">hyperlinks</span>

PagesEdit → Substitutions → uncheck Smart <span class="no-hyphens">Links</span>
