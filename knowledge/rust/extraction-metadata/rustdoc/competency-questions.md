# Competency Questions for Rustdoc

## Definitional (What is X?)
1. What is rustdoc?
2. What are doc comments (///, //!)?
3. What are doc tests?
4. What is the #[doc] attribute?
5. What are rustdoc lints?
6. What is the rustdoc search feature?
7. What are scraped examples?

## Procedural (How do I do X?)
8. How do I run rustdoc?
9. How do I write documentation comments?
10. How do I write and run doc tests?
11. How do I hide code in doc tests?
12. How do I link to other items in documentation?
13. How do I document re-exports?
14. How do I read and navigate rustdoc output?
15. How do I use rustdoc's command-line options?

## Diagnostic (What distinguishes X from Y?)
16. What is the difference between /// and //! doc comments?
17. What is the difference between doc tests and unit tests?
18. What is the difference between stable and unstable rustdoc features?
