# Competency Questions for The Rust Programming Language

## Definitional (What is X?)
1. What is ownership in Rust?
2. What are structs and how are they used?
3. What are enums and pattern matching?
4. What is Rust's module system?
5. What are Rust's common collections (Vec, String, HashMap)?
6. What is Rust's error handling model?
7. What are generics, traits, and lifetimes?
8. What is Rust's testing framework?
9. What are closures and iterators?
10. What are smart pointers (Box, Rc, RefCell)?
11. What is Rust's concurrency model?
12. What is async/await in Rust?
13. What are advanced Rust features (unsafe, traits, types, macros)?

## Procedural (How do I do X?)
14. How do I use ownership and borrowing?
15. How do I define and use structs?
16. How do I use enums with match?
17. How do I organize code with modules?
18. How do I handle errors with Result and Option?
19. How do I write generic functions and implement traits?
20. How do I write tests in Rust?
21. How do I use closures and iterators?
22. How do I do concurrent programming?

## Diagnostic (What distinguishes X from Y?)
23. What is the difference between String and &str?
24. What is the difference between Box, Rc, and Arc?
25. What is the difference between traits and interfaces?
