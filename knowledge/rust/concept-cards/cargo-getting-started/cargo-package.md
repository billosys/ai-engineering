# Quick Definition

A Cargo package is a unit of Rust code organization that contains a `Cargo.toml` manifest and one or more crates. It is the top-level artifact that Cargo creates, builds, and manages.

# Core Definition

The Cargo Getting Started guide introduces a package as something generated by `cargo new` and compiled by Cargo. The source describes `cargo new` as generating "a new package" and demonstrates "its ability to compile the crate within the package" (Ch. 2: First Steps with Cargo). A package contains a `Cargo.toml` manifest with metadata "that Cargo needs to compile your package" and a `src/` directory containing the crate's source code. The term "package" is highlighted as a key glossary term in the source text.

# Prerequisites

This is a foundational concept with no prerequisites within this source.

# Key Properties

1. Created by the `cargo new` command
2. Contains a `Cargo.toml` manifest at its root
3. Contains a `src/` directory with source code
4. Contains one or more crates (binary or library)
5. Has a name and version defined in the manifest
6. Is the unit that Cargo builds and manages

# Construction / Recognition

## To Create a Package:
1. Run `cargo new <package_name>`
2. The resulting directory is a package

## To Recognize a Package:
1. Look for a `Cargo.toml` file at the directory root
2. Look for a `src/` directory containing Rust source files
3. The presence of both indicates a Cargo package

# Context & Application

A package is the fundamental unit of code organization in the Cargo ecosystem. When a developer says "I'm working on a Rust project," they typically mean they are working within a Cargo package. The package concept sits at a level above crates -- a package contains crates, and the manifest describes the package as a whole including its dependencies.

# Examples

**Example 1** (Ch 2): Creating a package named `hello_world`:
```console
$ cargo new hello_world
```

**Example 2** (Ch 2): The package structure generated by `cargo new`:
```
hello_world/
├── Cargo.toml
└── src
    └── main.rs
```

**Example 3** (Ch 2): The manifest identifies the package by name and version:
```toml
[package]
name = "hello_world"
version = "0.1.0"
edition = "2024"
```

# Common Errors

- **Error**: Modifying the package directory structure by moving `Cargo.toml` into a subdirectory.
  **Correction**: `Cargo.toml` must remain at the package root. Cargo expects the conventional layout.

# Common Confusions

- **Confusion**: Using "package" and "crate" interchangeably.
  **Clarification**: A package is the container (directory + manifest); a crate is the compilable unit of code within the package. A package contains crates.

- **Confusion**: Thinking a package can only contain one crate.
  **Clarification**: While the Getting Started guide shows a single-crate package, packages can contain multiple crates (not elaborated in this source).
