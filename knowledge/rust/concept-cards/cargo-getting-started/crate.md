# Quick Definition

A crate is the compilable unit of Rust code within a Cargo package. It is the code that the Rust compiler actually processes, as distinguished from the package which is the broader organizational container.

# Core Definition

The Cargo Getting Started guide introduces "crate" as a key glossary term, describing Cargo's "ability to compile the crate within the package" (Ch. 2: First Steps with Cargo). The source distinguishes crate from package: the package is generated by `cargo new`, while the crate is what gets compiled within it. Crates come in two forms -- binary crates and library crates -- as indicated by the source's reference to the generated "hello world" program as "a binary crate."

# Key Properties

1. A crate is the unit of compilation within a package
2. Crates exist in two forms: binary and library
3. A crate resides in the `src/` directory of a package
4. The Rust compiler operates on crates, not packages
5. A binary crate has a `main.rs` entry point; a library crate has a `lib.rs` entry point

# Construction / Recognition

## To Recognize a Crate:
1. Look inside a Cargo package's `src/` directory
2. If `src/main.rs` exists, there is a binary crate
3. If `src/lib.rs` exists, there is a library crate
4. The crate is the source code that Cargo compiles

# Context & Application

The crate is a fundamental concept in the Rust ecosystem. In the context of the Getting Started guide, it appears as the thing Cargo compiles when you run `cargo build`. Understanding the crate concept is important because the Rust ecosystem uses "crate" as the standard term for a unit of compiled code -- crates.io (the Rust package registry) is named after this concept. The Getting Started guide introduces crates only briefly, with fuller treatment expected in later Cargo documentation.

# Common Errors

- **Error**: Creating Rust source files outside the `src/` directory and expecting Cargo to compile them.
  **Correction**: Place crate source code in the `src/` directory where Cargo expects to find it.

# Common Confusions

- **Confusion**: Using "crate" and "package" as synonyms.
  **Clarification**: A package is the organizational container (directory + `Cargo.toml`); a crate is the compilable code unit within a package. Cargo generates packages; the compiler compiles crates.

- **Confusion**: Thinking "crate" refers only to third-party dependencies.
  **Clarification**: Your own project code is also a crate. Every Cargo package contains at least one crate.
