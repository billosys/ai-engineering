# Quick Definition

The Cargo manifest is the `Cargo.toml` file at the root of every Cargo package. It contains all the metadata Cargo needs to compile the package, including its name, version, edition, and dependencies.

# Core Definition

The source explicitly defines the manifest: "This is called a manifest, and it contains all of the metadata that Cargo needs to compile your package" (Ch. 2: First Steps with Cargo). The manifest is a TOML-formatted file named `Cargo.toml` that sits at the root of a package directory. It is generated automatically by `cargo new` with default values.

# Prerequisites

This is a foundational concept with no prerequisites within this source.

# Key Properties

1. Named `Cargo.toml` (case-sensitive)
2. Located at the root of a Cargo package directory
3. Written in TOML format
4. Contains a `[package]` section with name, version, and edition
5. Contains a `[dependencies]` section for declaring external dependencies
6. Automatically generated by `cargo new`
7. Required for Cargo to compile the package

# Construction / Recognition

## To Create a Manifest:
1. Run `cargo new <name>` -- the manifest is generated automatically
2. Or create `Cargo.toml` manually at the package root

## To Recognize a Manifest:
1. Look for a file named `Cargo.toml` at the directory root
2. It should contain at minimum a `[package]` section with `name` and `version`

# Context & Application

The manifest is the central configuration file for any Cargo package. It defines the package identity (name, version), the Rust edition to use, and all dependencies. Cargo reads this file to determine how to build the package. Every Cargo operation depends on the manifest being present and correctly formatted.

# Examples

**Example 1** (Ch 2): The generated `Cargo.toml` for the `hello_world` package:
```toml
[package]
name = "hello_world"
version = "0.1.0"
edition = "2024"

[dependencies]
```

**Example 2** (Ch 2): The manifest's location in the package structure:
```
hello_world/
├── Cargo.toml    <-- the manifest
└── src
    └── main.rs
```

# Common Errors

- **Error**: Misnaming the file (e.g., `cargo.toml` with lowercase 'c').
  **Correction**: The file must be named exactly `Cargo.toml` with an uppercase 'C'.

- **Error**: Removing the `[package]` section or the `name`/`version` fields.
  **Correction**: These fields are required. Cargo will not compile without them.

# Common Confusions

- **Confusion**: Thinking `Cargo.toml` is only for dependencies.
  **Clarification**: The manifest contains package identity metadata (name, version, edition) in addition to dependencies. The source shows the `[package]` section as the primary content.

- **Confusion**: Confusing `Cargo.toml` (the manifest you edit) with `Cargo.lock` (the lockfile Cargo generates).
  **Clarification**: `Cargo.toml` is the manifest you maintain; `Cargo.lock` is auto-generated to lock dependency versions. (Note: `Cargo.lock` is not mentioned in this source.)
