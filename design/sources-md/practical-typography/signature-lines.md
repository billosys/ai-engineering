signature linesType a sequence of underscores

A *signature line* is a horizontal line aligned with adjacent <span class="no-hyphens">text.</span>

Typography purists avoid accomplishing anything by holding down keys on the keyboard. But in this case it’s the simplest solution. To make a signature line, hold down the underscore key (shift + hyphen) until you get the length you <span class="no-hyphens">need.</span>

\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_

A rival school of thought suggests you should type a series of <a href="word-spaces.html" class="xref">word spaces</a> and format them with <a href="underlining.html" class="xref">underlining</a>.

\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_\_

To make a signature block that won’t break across pages, see <a href="keep-lines-together.html" class="xref">keep lines together</a>.

Same thing, right? Not quite. There are three good reasons to prefer underscores to underlined word <span class="no-hyphens">spaces.</span>

1.  If you need to quickly rid a document of underlining, you might want to select all the text and then uncheck the underlining option. But this will wreck signature lines made out of underlined word spaces—they will <span class="no-hyphens">disappear.</span>

2.  If you need to quickly ensure you only have <a href="one-space-between-sentences.html" class="xref">one space between sentences</a>, you might want to search for and replace any double spaces. But this will also wreck signature lines made out of word spaces—by partially deleting <span class="no-hyphens">them.</span>

3.  <a href="rules-and-borders.html" class="xref">Rules and borders</a> can also work, but will likewise be lost when copying and <span class="no-hyphens">pasting.</span>

    Underscore characters don’t depend on formatting, so they will look the same no matter where they’re copied and pasted. Underlined word spaces may <span class="no-hyphens">not.</span>

You should depart from this rule only if the font you’re using has an underscore character that doesn’t form a solid line when used in sequence. That’s how underscores are supposed to work, but some fonts are uncooperative. If you see gaps between the underscores, either use underlined word spaces, or use the underscores from a different font. No one will <span class="no-hyphens">know.</span>

But <a href="nonbreaking-spaces.html" class="xref">nonbreaking spaces</a> will <span class="no-hyphens">work.</span>
