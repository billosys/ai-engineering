system fontsAvoid if you can, choose wisely if you can’t

System fonts are the fonts already installed on your computer. Some are better than others. In printed documents they present three <span class="no-hyphens">problems.</span>

1.  *Many system fonts aren’t good*. The Windows and Mac OS libraries have improved, but they’re still minefields of awful fonts. I won’t name names, but my least favorite rhymes with <span class="no-hyphens">Barial.</span>

2.  In one square inch, an LCD screen that displays 100 dots per inch has less than 3% of the resolution of a laser printer with 600 dots per inch, so rendering a font accurately is much more difficult. See <a href="screen-reading-considerations.html" class="xref">screen-reading considerations</a>.

    *Many system fonts have been optimized for the screen, not print*. This comes at the cost of design details, which have been sanded off because they don’t reproduce well on screen (e.g., Georgia, Verdana, Cambria, and Calibri). Screen-optimized fonts look clunky on the printed <span class="no-hyphens">page.</span>

    Compare the two fonts above. In basic appearance, they’re similar. But Georgia was optimized for the screen; Miller was optimized for print. See the <span class="no-hyphens">difference?</span>

3.  *All system fonts are overexposed*. Because these fonts are included with billions of computers, they’re used all the time. Not every typography project demands novelty. But if yours does, look elsewhere. For instance, please don’t adopt the slogan “A Design Firm Unlike Any Other” and then set it in <span class="no-hyphens">Helvetica.</span>

*WARNING*  
This chart is offered only as a harm-reduction device. In the long term, a diet of system fonts can be harmful to your health. My official advice remains the same: professional writers should use professional fonts. In the pages following, I suggest professional alternatives to the most common system <span class="no-hyphens">fonts.</span>

If you’re limited to system fonts, consult this chart and choose wisely. For print, the A list is best. For screen display, like presentations and websites, the A and B lists are fine. They’re also suitable for sharing draft documents. Avoid the C list if you can. F list, <span class="no-hyphens">kapu.</span>

Fonts plausible for <a href="body-text.html" class="xref">body text</a> are marked with <span style="font-family: concourse-t3">★</span>. Others are usable for special purposes (for instance, <a href="letterhead.html" class="xref">letterhead</a>).

This chart includes all the common Windows and Mac system fonts, plus the Microsoft Office fonts. System configurations differ, so not every font will be on your <span class="no-hyphens">computer.</span>

These rankings represent a blend of practical and aesthetic considerations, not absolute merit. Some fonts on the F list aren’t bad. They’re just inapt for professional writing. Similarly, some fonts on the A list are not my favorites, but they’re reasonably <span class="no-hyphens">useful.</span>

<div class="btw-title">

by the way

</div>

- Adobe uses its own text-rendering technology in Acrobat so that PDF documents display the same way on screen regardless of the underlying operating <span class="no-hyphens">system.</span>

- “My PDF will probably be read on screen. Shouldn’t I use a screen-optimized system font?” No. In Windows, certain system fonts (e.g., Georgia, Calibri) have been optimized by Microsoft for user-interface purposes. This is accomplished with hinting, which is extra software code stored in the font itself. Windows relies on this hinting when it draws text on screen (e.g., in Microsoft Word, or in a web <span class="no-hyphens">browser).</span>

  Due to advances in display technology, the screen-legibility advantage of these particular system fonts is rapidly evaporating—see <a href="screen-reading-considerations.html" class="xref">screen-reading considerations</a> for why. Furthermore, any PDF could also end up being printed. Therefore, as a rule, you’re better off using print-optimized fonts for <span class="no-hyphens">PDFs.</span>

- “But if I use a print-optimized professional font in my PDF instead of a system font, my readers probably won’t have the same font installed.” Right. But it doesn’t matter. When you generate a PDF, your fonts are embedded in the PDF to preserve the <span class="no-hyphens">formatting.</span>

- This is not true, however, on the web. Web browsers use the text rendering of the operating system. Thus, in Windows browsers, screen-optimized system fonts have traditionally held an advantage, because they look good and they’re already installed. (Indeed, the Microsoft fonts Georgia and Verdana were specifically created for web use.) But this advantage is rapidly fading with the advent of screen-optimized webfonts and the general shift toward higher-resolution screens. Still, for now, using professional fonts on a website requires a little more legwork than it does in <span class="no-hyphens">PDF.</span>

- Yes, I dislike Arial more than Comic Sans. Though it’s the undisputed king of the <a href="goofy-fonts.html" class="xref">goofy fonts</a>, Comic Sans is at least honest about what it is. But Arial is merely a bland, zero-calorie Helvetica <span class="no-hyphens">substitute.</span>

  <div class="gap" style="height:0.3em">

  </div>

  For many, the two are indistinguishable. But for typographers, Arial contains none of the consistency and balance that makes Helvetica <span class="no-hyphens">successful.</span>

  Still, the main issue is overuse. After decades as a system font, Arial has achieved a ubiquity that rivals <a href="times-new-roman-alternatives.html" class="xref">Times New Roman</a>. And like Times New Roman, Arial is permanently associated with the work of people who will never care about <span class="no-hyphens">typography.</span>

  You’re not one of those people. So use Avenir. Use Franklin Gothic. Use Gill Sans. Use one of the fonts listed in <a href="helvetica-and-arial-alternatives.html" class="xref">Helvetica and Arial alternatives</a>. Or use something completely different. But don’t use Arial. It’s the sans serif of last <span class="no-hyphens">resort.</span>
