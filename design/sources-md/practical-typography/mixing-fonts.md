mixing fontsLess is more

Enthusiasm for fonts often leads to enthusiasm for multiple fonts, and then the question: “How do I get better at mixing fonts in a <span class="no-hyphens">document?”</span>

Mixing fonts is like mixing patterned shirts and ties—there aren’t immutable rules. Some people have a knack for it; some <span class="no-hyphens">don’t.</span>

Keep these principles in <span class="no-hyphens">mind:</span>

1.  Mixing fonts is never a requirement—it’s an option. You can get plenty of mileage out of one font using variations based on <a href="point-size.html" class="xref">point size</a>, <a href="bold-or-italic.html" class="xref">bold or italic</a>, <a href="small-caps.html" class="xref">small caps</a>, and so <span class="no-hyphens">on.</span>

2.  The rule of diminishing returns applies. Most documents can tolerate a second font. Few can tolerate a third. Almost none can tolerate four or more. (If you’re making a <a href="presentations.html" class="xref">presentation</a>, consider all your slides to be part of one <span class="no-hyphens">document.)</span>

3.  See <a href="bold-or-italic.html" class="xref">bold or italic</a> for the difference between serif and sans serif <span class="no-hyphens">fonts.</span>

    You can mix any two fonts that are identifiably different. If you’ve heard that you can only mix a serif font with a sans serif font, it’s not true. On the contrary, much like mixing <a href="color.html" class="xref">colors</a>, lower contrast between fonts can be more effective than higher <span class="no-hyphens">contrast.</span>

    Look at any American newspaper—typically, the <a href="body-text.html" class="xref">body text</a> and the headlines are both in serif fonts, but different <span class="no-hyphens">ones.</span>

4.  Font mixing is most successful when each font has a consistent role in the document. In a <a href="research-papers.html" class="xref">research paper</a>, try one font for body text and one font for <a href="headings.html" class="xref">headings</a>. Or try one font for things in the center of the document (body text and headings) and one font for things at the edges (line numbers, footer, and other miscellany). Or in <a href="bulleted-and-numbered-lists.html" class="xref">bulleted and numbered lists</a>, try one font for the bullet or number and one font for the text of the list item—a technique I use throughout this <span class="no-hyphens">book.</span>

5.  It rarely works to have multiple fonts in a single paragraph. Better to restrict yourself to one font per paragraph, and change fonts only at paragraph <span class="no-hyphens">breaks.</span>

6.  Though I’m typically reluctant to endorse rote methods, this one works reliably: combine fonts by the same font designer. For instance, pairings of <a href="fontrec/atlas.html" class="xref">Atlas</a> and <a href="fontrec/lyon.html" class="xref">Lyon</a> (both designed by Kai Bernau), <a href="fontrec/alright-sans.html" class="xref">Alright Sans</a> and <a href="fontrec/harriet.html" class="xref">Harriet</a> (by Jackson Cavanaugh) or and (by <span class="no-hyphens">me).</span>
