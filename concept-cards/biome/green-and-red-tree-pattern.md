# Quick Definition
The Green and Red tree pattern is a library design pattern, originating from Microsoft's Roslyn compiler, that separates syntax tree storage (the immutable "green" tree) from navigation and parent-aware access (the "red" tree facade).

# Core Definition
The architecture of Biome's parser is built on an internal fork of rowan, a library that implements the Green and Red tree pattern. In this pattern, the CST is never directly accessible by design; a developer reads its information using the Red tree, through a number of APIs that are autogenerated from the grammar of the language. The pattern originates from Microsoft's Roslyn .NET compiler platform, described by Eric Lippert as "persistence, facades, and Roslyn's red-green trees."

# Key Properties
1. **Two-layer architecture** -- Separates the underlying data (green tree) from the developer-facing API (red tree)
2. **Rowan implementation** -- Biome uses an internal fork of the rowan library (originally from rust-analyzer)
3. **Indirect access** -- The CST is accessed through the Red tree, not directly
4. **Autogenerated APIs** -- Developer-facing APIs are generated from language grammars
5. **Roslyn lineage** -- Pattern originated in Microsoft's Roslyn compiler

# Construction / Recognition
The Green and Red tree pattern is an implementation detail of Biome's parser. Developers interact with it through autogenerated APIs rather than constructing or manipulating trees directly.

# Context & Application
This pattern enables efficient representation of syntax trees that can be shared and reused while providing a convenient navigation API. The rowan library, originally developed for rust-analyzer, brings this pattern to the Rust ecosystem. Biome's internal fork allows customization for its specific needs.

# Common Confusions
1. **Green vs. Red** -- The "green" tree is the immutable underlying data; the "red" tree is the facade that provides parent-aware navigation. Developers interact with the red tree.
2. **rowan vs. Roslyn** -- rowan is a Rust implementation of the pattern; Roslyn is Microsoft's original .NET implementation. Biome uses a fork of rowan.
