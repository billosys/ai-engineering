# Quick Definition

Build options are the top-level settings in `_cobalt.yml` that control how Cobalt discovers source files, which files are treated as pages, whether drafts are included, which files are ignored, and where output is written.

# Core Definition

Build options are the set of configuration settings in `_cobalt.yml` that control the build process. They include: `source` (the directory where content lives, relative to `_cobalt.yml`, default: `"."`), `template_extensions` (file extensions that mark a file as a page rather than an asset, default: `["md", "liquid"]`), `include_drafts` (whether draft posts are generated by default, default: `false`), `ignore` (gitignore-style patterns for files to exclude, relative to source), and `destination` (the output directory, relative to `_cobalt.yml`, default: `"_site"`). These options can also be overridden via command-line flags. (Source: Cobalt Configuration documentation, "Build options" section)

# Prerequisites

- **[Cobalt Configuration File](/concept-cards/cobalt/cobalt-configuration-file.md)** -- Build options are a section within `_cobalt.yml`.

# Key Properties

1. **`source`**: String, directory where content lives, default `"."`.
2. **`template_extensions`**: List of strings, file extensions that mark pages (default: `["md", "liquid"]`).
3. **`include_drafts`**: Boolean, whether drafts are generated (default: `false`); overridable via `--drafts` CLI flag.
4. **`ignore`**: List of strings, gitignore-style patterns for files to exclude.
5. **`destination`**: String, output directory (default: `"_site"`).
6. **All paths are relative**: Paths are relative to the location of `_cobalt.yml`.

# Construction / Recognition

## To Construct/Create:
1. In `_cobalt.yml`, add top-level keys:
   ```yaml
   source: "."
   template_extensions:
     - md
     - liquid
   include_drafts: false
   ignore:
   destination: _site
   ```

## To Identify/Recognize:
1. Look for the top-level keys `source`, `template_extensions`, `include_drafts`, `ignore`, and `destination` in `_cobalt.yml`.

# Context & Application

- **Typical contexts**: Setting up a Cobalt project's build pipeline.
- **Common applications**: Changing the output directory, adding custom template extensions, ignoring specific files, enabling draft inclusion for development.

# Examples

**Example 1** (source: Configuration documentation): Default build options:
```yaml
source: "."
template_extensions:
  - md
  - liquid
include_drafts: false
ignore:
destination: _site
```

**Example 2** (source: Configuration documentation): Unignoring the `.well-known/` folder for LetsEncrypt:
```yaml
ignore:
  - "!.well-known"
  - "!.well-known/**/*"
```

# Common Errors

- **Error**: Using absolute paths for `source` or `destination`.
  **Correction**: These paths are relative to the location of `_cobalt.yml`. Use relative paths.

- **Error**: Forgetting to add a custom extension (e.g., `.html`) to `template_extensions` and wondering why those files are copied as assets instead of processed.
  **Correction**: Add the extension to `template_extensions` to have Cobalt treat it as a page.

# Common Confusions

- **Confusion**: Thinking `ignore` patterns are relative to the project root.
  **Clarification**: `ignore` patterns are relative to the `source` directory, not the project root (unless source is `.`).
