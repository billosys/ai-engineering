<div id="doc">

ordinalsNo superscripts

Ordinals are numbers that express position in a series. By default, your word processor will convert ordinals into <span class="no-hyphens">superscripts.</span>

|                                                |                 |
|------------------------------------------------|-----------------|
| 1<sup>st</sup> 2<sup>nd</sup> 304<sup>th</sup> | superscripts    |
| 1st 2nd 304th                                  | no superscripts |

Superscripted ordinals are tiny and hard to read. The best policy is to avoid them. To do this, you have to adjust your word processor’s default <span class="no-hyphens">behavior.</span>

<div class="howto">

<div class="howto-name">

How to turn off superscripted ordinals

</div>

WordFile → Options → Proofing → AutoCorrect Options → AutoFormat As You Type → uncheck Ordinals (1st) with <span class="no-hyphens">superscript</span>

Mac OS WordWord → Preferences → AutoCorrect → AutoFormat As You Type → uncheck Ordinals (1st) with <span class="no-hyphens">superscript</span>

PagesDoes not superscript <span class="no-hyphens">ordinals</span>

</div>

<div class="btw">

<div class="btw-title">

by the way

</div>

- In HTML, nothing will be superscripted unless you put it in a `<sup>` tag. But there is no reason to do <span class="no-hyphens">so.</span>

- Some fancy professional fonts (see <a href="font-recommendations.html" class="xref">font recommendations</a>) include specially designed superscript alphabets and <a href="alternate-figures.html" class="xref">alternate figures</a> that remain legible at small sizes. Those are fine to use in ordinals, but they’re typically made available as <a href="opentype-features.html" class="xref">OpenType features</a>. So you’ll still want to turn off any automatic ordinal shrinking performed by your writing <span class="no-hyphens">tool.</span>
