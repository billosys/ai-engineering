trademark & copyright symbolsDon’t use alphabetic substitutes

Your keyboard includes a trademark symbol (™), a registered trademark symbol (®), and a copyright symbol (©).

|     |                      | Windows  | Mac OS     | HTML      |
|-----|----------------------|----------|------------|-----------|
| ™   | trademark            | alt 0153 | option + 2 | `&trade;` |
| ®   | registered trademark | alt 0174 | option + r | `&reg;`   |
| ©   | copyright            | alt 0169 | option + g | `&copy;`  |

When you need these symbols, use them. Don’t use goofy alphabetic approximations like *(TM)* or *(c)*—another <a href="typewriter-habits.html" class="xref">typewriter habit</a>.

|                        |       |
|------------------------|-------|
| (c) 2026 MegaCorp (TM) | wrong |
| © 2026 MegaCorp™       | right |

Trademark symbols are set as superscripts—smaller characters positioned above the baseline of the text (*Roxy’s Tacos*™, *Caring Is Our Business*®). If you use proper trademark symbols, they’ll appear at the right size and height. No space is needed between the text and the trademark <span class="no-hyphens">symbol.</span>

Copyright symbols appear in line with the text (*© 1999*). Use a <a href="nonbreaking-spaces.html" class="xref">nonbreaking space</a> between the copyright symbol and the year to ensure the two don’t end up on different lines or <span class="no-hyphens">pages.</span>

Must you put a word space after the copyright symbol? No, but it makes good sense semantically. The © directly replaces the word copyright, so it should be spaced like any other word. Trademark symbols, on the other hand, are more like little barnacles of legalese—immovable but mostly ignored. To think of it another way, if you were reading aloud the phrase “© 2026 MegaCorp™”, you’d likely pronounce the © but not the <span class="no-hyphens">™.</span>

<div class="btw">

<div class="btw-title">

by the way

</div>

- Many word processors automatically substitute a symbol when you type (TM), (R), or (c). Consider turning off this feature, because references like *Section 12(c)* can be silently converted to *Section 12©*. The spell-checker won’t detect this error, and it’s easy to overlook while <span class="no-hyphens">editing.</span>

- *Copyright © 2026* is redundant. Word or symbol—not <span class="no-hyphens">both.</span>

- I always use the nonitalicized versions of these marks, even if the adjacent text is italicized. I think it looks better. But that’s a preference, not a <span class="no-hyphens">rule.</span>
